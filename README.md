# YLG48 Wife Lottery - AstrBot 插件

一个QQ群随机抽奖机插件，支持多群部署、许愿系统、排名统计等功能。

## 🚀 快速开始

### 1. 安装依赖
```bash
pip install -r requirements.txt
```

### 2. 启动 AstrBot
```bash
astrbot run
```

### 3. 配置插件
打开 AstrBot 管理面板（http://localhost:6099），找到 YLG48 Wife Lottery 插件，配置以下必需项：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| target_group | 目标群号（逗号分隔多个） | 764687233 |
| db_host | 数据库地址 | localhost |
| db_port | 数据库端口 | 3306 |
| db_user | 数据库用户 | root |
| db_password | 数据库密码 | password |
| db_name | 数据库名 | lottery |

保存并重启插件。

## 📋 使用指令

默认指令（可在配置中修改）：

| 指令 | 功能 |
|------|------|
| 来个老婆 | 随机抽取一个成员 |
| 换个老婆 | 更换老婆（2小时冷却） |
| 我的老婆 | 查询个人统计数据 |
| 许愿 [名字] | 许愿特定成员 |
| 排名 | 查看群内排名 |

## ⚙️ 核心配置

### 指令配置
```
lottery: 来个老婆,换个老婆      （抽奖指令，逗号分隔）
data: 我的老婆                  （数据查询指令）
wish: 许愿                      （许愿指令）
rank: 排名                      （排名指令）
```

### 功能配置
```
bring_threshold: 80             （"带走"所需的最低情愫值）
anti_recall_qq: 123456,789012   （防撤回监控的QQ号，逗号分隔）
```

### 消息文本配置

所有提示消息都可在配置中修改：

- `firstCoinMsg` - 每日第一提示
- `lotteryOut` - 抽奖输出格式
- `dataOut` - 数据查询输出格式
- `dataVoidOut` - 无老婆提示
- `senseQueenMsg` - 情愫王信息格式
- `wishSuccess` - 许愿成功提示
- `errorMsg` - 错误提示
- 等等...（详见 _conf_schema.json）

## 💎 功能说明

### 情愫值系统
每次抽奖随机获得 0-100% 的情愫值，根据值给出推荐：
- 100%+ → 原地结婚
- 89-99% → 最佳拍档
- 79-89% → 带回家吧
- 69-79% → 约会去吧
- 19-69% → 再努努力
- 19%以下 → 下辈子吧

### 许愿系统
- 发送 `许愿 成员名` 开始许愿
- 有 10 次机会在抽奖中匹配目标
- 成功时情愫值+10，获得 20 币奖励

### 多群支持
配置中用逗号分隔多个群号，可同时在多个群运行，数据独立管理。

### 防撤回功能
监控指定 QQ 的消息，被撤回时自动复原。

### 排名系统
统计群内"带走老婆"（情愫值≥阈值）的次数排名。

## 🗄️ 数据库

## 🐛 常见问题

### Q: 如何修改抽奖指令？
A: 在管理面板中修改 `lottery` 配置项（逗号分隔多个指令）。

### Q: 如何修改提示文本？
A: 在管理面板中修改对应的消息配置项（如 `lotteryOut`、`dataOut` 等）。

### Q: 怎样支持多个群？
A: 在 `target_group` 配置项中用逗号分隔群号，如 `123456,789012,345678`。

### Q: 防撤回功能如何使用？
A: 在 `anti_recall_qq` 中填入要监控的 QQ 号（逗号分隔），这些 QQ 的消息被撤回时会自动复原。

### Q: 数据库连接失败怎么办？
A: 检查 MySQL 是否运行，确认数据库配置正确（host、user、password、db_name）。
